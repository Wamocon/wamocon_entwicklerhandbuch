KAP 08

Entwicklungsprozess

Von der Produktidee bis zum Go-live – der vollständige Schritt-für-Schritt-Prozess.

  • Anforderungsdokument erstellen & in OneDrive ablegen (Welle / App-Ordner)
  • Nach Freigabe: Supabase Cloud & Strato-Domain per E-Mail beantragen
  • Entwicklungsstart: Template Repo → dev-Branch → Anforderungsdokument in Copilot
  • Copilot erstellt Entwicklungsplan (Schema, Komponenten, API, Meilensteine)
  • Iterative Implementierung mit Copilot entlang des Plans
  • Landing Page generieren via KI-Workflow (bevorzugt GitHub Pages)

KAP 09

Jira & Aufgabenmanagement

Aufgabenverwaltung mit Jira – optional bei Wellen-Apps, verpflichtend bei großen Projekten außerhalb des Wellen-Rhythmus.

  • Jira optional für Wellen-Apps, verpflichtend bei Großprojekten
  • Ticket-Typen: Story, Task, Bug, Sub-Task
  • Status-Workflow: Backlog → To Do → In Progress → In Review → Done
  • ARGUS Review Pipeline: Multi-Agent Review für Entwicklungs-Artefakte

KAP 10

Supabase Cloud Projekt

Datenbankprojekte beantragen, einrichten und nutzen – WAMOCON nutzt Supabase Pro.

  • Beantragung per E-Mail (kein Self-Service) – direkt nach Anforderungsfreigabe
  • .env.local: SUPABASE_URL, ANON_KEY, SERVICE_ROLE_KEY, DATABASE_URL
  • Migrationen: Copilot erstellt & führt sie direkt in Supabase Cloud aus
  • Row Level Security (RLS): Pflicht für alle Tabellen
  • Lokale Instanz via localSupabaseDB als Fallback (max. 2 parallel)
  • Wechsel lokal → Cloud nach Projektgenehmigung

KAP 11

Vercel Deployment

Deployment aller TypeScript-Apps über Vercel mit dem GitHub Actions Workflow aus dem Template Repo.

  • Vercel Hobby Plan – Deployment über GitHub Actions aus template_repo
  • Einmaliges Setup: Repo öffentlich → Vercel importieren → VERCEL_PROJECT_ID als Secret
  • Push auf dev → Preview-Deploy, Merge auf main → Produktions-Deploy
  • Landing Pages bevorzugt auf GitHub Pages (kostenlos, kein Build-Step)
  • Domain-Konfiguration: Strato (A/CNAME) + Vercel (SSL automatisch)

KAP 12

Sicherheit & Code-Qualität

Entwicklungsstandards für sicheren und qualitativ hochwertigen Code.

  • RLS auf allen Supabase-Tabellen – verpflichtend

  • TypeScript Strict Mode, ESLint, Pre-Commit Hooks
  • Dependabot, npm audit, GitHub Secret Scanning
  • Performance: next/image, Paginierung, Datenbankindizes
