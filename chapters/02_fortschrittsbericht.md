# Kapitel 02 – Fortschrittsbericht und Live-Dokumentation

## Zielbild

Die Erstellung und Pflege des Entwicklerhandbuchs soll für Nutzer nachvollziehbar dokumentiert werden.

## Einschränkung

Ein Copilot-Agent kann keine dauerhaft autonome, minütliche Chat-Aktualisierung garantieren. Deshalb wird stattdessen sofort ein dokumentierter Status-Stand bereitgestellt.

## Verbindliche Inhalte eines Status-Reports

| Feld | Inhalt |
| --- | --- |
| Timestamp | Zeitpunkt der letzten Aktualisierung in UTC |
| Ablaufplan | Welche Arbeitspakete existieren |
| To-do-Status | Offen, in Arbeit oder erledigt |
| Abrufweg | Wie der Nutzer weitere Updates erhält |

## Manueller Abruf

1. Nutzer fragt im Chat nach einem neuen Fortschrittsbericht.
2. Der Agent ergänzt den Status und nennt den aktuellen Timestamp.
3. Repo und GitHub Pages werden aktualisiert oder geprüft.
